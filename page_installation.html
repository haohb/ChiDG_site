Installation

This page details the information required to successfully build the ChiDG project. One should be sure to first address the required dependencies of the project, which are detailed below. Also, a recent compiler version should be used. The following steps outline the process for successfully building ChiDG.

Obtaining ChiDG

ChiDG is hosted in a git repository on www.gitlab.com and may be cloned using either ssh or https protocols as

git clone git@gitlab.com:nwukie/ChiDG.git ChiDG
git clone https://gitlab.com/nwukie/ChiDG.git ChiDG

ChiDG is currently in a pre-release state. As such, public read-access to the above repository is restricted.

Compile MPICH

When compiling mpich, the following procedure is used to set the relevant compilers and configure mpich to be built.

The following configuration command is used in the root mpich directory to set up the build:

./configure --prefix=/usr/local/mpich/gnu --enable-fortran=all CC=gcc CXX=g++ F77=gfortran FC=gfortran 2>&1 | tee c.txt

The CC, CXX, F77, and FC variables set in the configure line are used to set the C, C++, and Fortran compilers.

Compile mpich with:

make 2>&1 | tee m.txt

Install mpich with:

make install 2>&1 | tee mi.txt

Set environment variables for MPICH

The cmake build script requires the MPI_ROOT environment variable to be set.

export MPI_ROOT=/usr/local/mpich

This must be set such that ${MPI_ROOT}/gnu or ${MPI_ROOT}/intel are valid; allowing the cmake script to select the correct installation based on the compiler vendor being used.

TecIO

Tecplot 360 is the primary visualization tool currently used for ChiDG. The TecIO libraries are distributed with the ChiDG project for multiple systems and should be linked accordingly without user intervention. If, for some reason, the packaged TecIO libraries are incompatible with your system, new libraries can be downloaded from the Tecplot website.

Compiling ChiDG

NOTE: ChiDG uses features of the most recent Fortran standard (F2008). Updated versions of Fortran compilers are required.

Successfully tested compiler versions:

  • gfortran 5.2

ChiDG uses the CMake build system. Out-of-source builds, as described here, are the only supported build method.

  • Change to the ChiDG root directory.
    cd ChiDG/
  • Create a new build directory.
    mkdir build
  • Change to the build directory.
    cd build
  • Execute the CMake command with options to configure the build
    cmake -DCMAKE_Fortran_COMPILER=gfortran-5.2 -DCMAKE_BUILD_TYPE=Release ..
  • Run make to build the ChiDG executable
    make
